## Pull Request Review Checklist

The following guidelines apply to all Pull requests raised for this repository. The PR author should ensure all evidence is submitted along with a relevant description. The reviewers should ensure that their review adheres to the review guidelines. If not, the author has the right to ignore feedback.

### Author

👥 **Reviewers**:

- Minimum of 2 reviewers.
- Is one reviewer from peer support?

📸 **Evidence**:
Provide supporting evidence of:

- AC being met (attach screenshots, videos, etc).
- Unit tests for the code in this PR.
- Evidence that all test suites pass.

* Include considerations made during development about:
  - Alternative coding approaches considered and reasons for the chosen implementation.
  * Accessibility considerations.
  * Performance considerations.
  * Maintainability considerations.

- (Frontend only)
  - Prettier/ESLint run without any raised issues.

### Reviewer

🔍 **Review Objectivity**:

- Does the PR meet the Acceptance Criteria (AC) stated in the ticket? Does the code fulfill its intended purpose?
- Are there valid concerns related to accessibility, maintenance, scalability, or performance?
- Does the code adhere to the YAGNI, KISS, and DRY principles?
- Ensure your comments are match what was agreed in the [ADRs](https://github.com/HCA-Eqtr/Sitecore-XM-Cloud-Spike/tree/main/documentation).

📝 **Conventional Comments**:

> [!IMPORTANT]
>
> - Please follow the comment conventions (inspired by [conventional comments](https://conventionalcomments.org/)).
>   - ❓ (Question) - For any question.
>   - 💭 (Suggestion) - For non-blocking suggestions.
>   - 🙌 (Praise) - To highlight impressive aspects.
>   - ✍ (Change) - For mandatory changes that need PR updates.

    - 🤓 (Nit-picking) - For minor, non-mandatory change requests that offer a new perspective or a learning opportunity.

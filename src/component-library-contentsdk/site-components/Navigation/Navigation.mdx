# Navigation

## Description

The Navigation site-component renders both the NavigationMobile and NavigationDesktop components.

It is intended to be used as the main navigation for a site.

### Shared behavior

- The prop type is shared between desktop and mobile navigation components.
- The navigation header transitions up out of view while the use is scrolling down, unless the nav drawer is open.
- The navigation header shows a different theme when closed vs. when opened.

### Prop structure

The expected prop type is structured so that the tabs represent the first level of navigation, and the content within each tab represents a list of child
components of template and variations.

#### Second level child template/variation (grid columns occupied on desktop):

- 'Main Navigation Links List'
- - 'single-narrow' (2)
- - 'single-wide' (3)
- - 'double' (6)
- - 'full' (12)
- - 'simple' (3)
- 'Navigation Content Block'
- - 'single' (3) (styled as a card)
- - 'double' (5)
- 'Navigation Blog Post Card' (3)
- 'Spacer' (1)

#### Mobile/Desktop Differences

- The desktop navigation consumes the content array within each tab in props, and renders them all as child components within each tab's drawer.
- The mobile navigation renders the child template 'Main Navigation Links List' as the secondary level of navigation, and extracts the `links` attribute from within to render as the tertiary level navigation.
- The mobile navigation will concatenate the primary and secondary navigation views in the case that there is only one secondary level 'Main Navigation Links List'

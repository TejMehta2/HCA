# Navigation

## Description

The Navigation site-component which renders both the NavigationMobile and NavigationDesktop components.

It is intended to be used as the main navigation for a site.

### Shared behavior

- The prop type is shared between desktop and mobile navigation components.
- The navigation header transitions up out of view while the use is scrolling down, unless the nav drawer is open.
- The navigation header shows a different theme when closed vs. when opened.

### Differences

The expected prop type is structured so that the tabs represent the first level of navigation, and the content within each tab represents a list of child components of type ('single-narrow' | 'single-wide' | 'double' | 'header' | 'card'). The child components can be added in any order.

- The desktop navigation consumes the content array within each tab in props, and renders them all as child components within each tab's drawer. 
- The mobile navigation renders only the child types ('single-narrow' | 'single-wide' | 'double') as the secondary level of navigation, and extracts the `links` attribute within to render as the tertiary level navigation.

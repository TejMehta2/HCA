# Pagination

## Description

The Pagination component is a wrapper component which splits the CardGrid component onto multiple pages

The page numbers shown are dynamic and are dependant on the current page of the user. The first and last page is always shown along with the previous and next page numbers

it includes a callback function which can be used to fetch the content to be shown on a particular page. This content is always rendered as a card, but card variation can be used

page count is a required prop for the initial render

It is intended to be used in the blog page and any generic content page

It can be composed like this:

```tsx
<Pagination
  theme="f"
  pageCount={14}
  callback={(newPage: number) => {
    return getPageContent(newPage);
  }}
  data={[
    <CardContent
      key={i}
      title={
        <Text variation="heading-1" tag="h4">
          {(i + 1).toString() + 'Cardiac care'}
        </Text>
      }
      bodyCopy={
        <Text variation="body-large" tag="p">
          There are over 1400 at The Portland, each year. Hear new mums sharing
          theirs
        </Text>
      }
      image={
        <Image
          src="/placeholders/multicard.jpg"
          alt="baby crying"
          width="363"
          height="243"
        />
      }
      link={
        <a href="#">
          <span>
            Read the <strong>Story</strong>
          </span>
        </a>
      }
    />,
    (... more cards)
  ]}
/>
```

# AddressFinder

## Description

The AddressFinder component's purpose is to allow the user to fill in their address in a form by either using an autocomplete or entering it manually.
Users are initially presented with the autocomplete option, where they can start typing their address and be presented a list of options, which if they click will appear as selected (at this point the autocomplete section is hidden). When selected the address chosen will be displayed and there are two buttons displayed below - `edit` and `remove`.

By clicking `edit` users are presented with the manual input fields with the address prefilled - allowing them to easily change it.

By clicking `remove`, the address is cleared and the address finder returns back to it's initial state displaying the autocomplete.

Diplayed below the manual address fields is a button `new search` which will clear any selected address and return the address finder back to it's initial state, this provides a way for the user to show the autocomplete option again if they had decided to fill the address in manually but changed their mind.

The story for AddressFinder has some mock results but the intention is for a request to be made in the parent component where AddressFinder is used and the result data be shared using the `addressResults` prop.

It is intended to be used on the payment form.

It can be composed like this:

```tsx
<AddressFinder
    addressResults: results, // address results from response
    searchAddress: (term) => {
      //  use the term to make request
    },
    chosenAddress: (address) => {
      //  address that has been set using the finder
    },
></AddressFinder>
```

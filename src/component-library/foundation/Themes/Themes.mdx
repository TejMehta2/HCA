# Themes

## Description

Themes expose a list of CSS custom variables in the cascade. The full list is at the bottom of this documentation.

## Assigning a theme

The current theme can be assigned as follows.

### Via SCSS mixin

``` scss
.my-component {
  @include apply-theme-a;
}
.my-component .my-nested-component {
  @include apply-theme-b
}
```

### Via React component

``` tsx
<Themes theme={'A-HCA-Main-Turquoise'}>
  <MyComponent />
</Themes>
```

Many of the site components inherit the theme from the cascade, as demonstrated in the stories.

The CSS custom properties of the theme can be consumed manually:

``` scss
.my-themed-component {
  background: var(--background);
  color: var(--text);

  .card {
    background: var(--card-background);
    color: var(--card-text);
  }
}
```

## Modes

Each theme has a light or dark mode, which determines colours automatically applied to nested components such as Cards.

We can also access and apply styles for light and dark modes via the `theme-mode` mixin e.g. to hide or show a specific logo based on the current theme's mode:

``` scss
.my-themed-component {
  background: var(--background);
  color: var(--text);

  .card {
    background: var(--card-background);
    color: var(--card-text);
  }

  @include theme-mode(light, dark) using ($mode) {
    @if $mode == light {
      .light-logo {
        display: block;
      }
      .dark-logo {
        display: none;
      }
    }
    @if $mode == dark {
      .light-logo {
        display: none;
      }
      .dark-logo {
        display: block;
      }
    }
  }
}
```

### Nested Themes

Themes can be nested, as shown below.

#### In React
``` tsx
<Themes theme={'A-HCA-Main-Turquoise'}>
  <ParentComponent>
    <h2>Here theme a</h2>
    <Themes theme={'B-HCA-Green'}>
      <h3>Here theme b</h3>
      <ChildComponent />
    </Themes>
  </ParentComponent>
</Themes>
```
#### In SCSS

``` scss
.wrapper {
  @include apply-theme-a;

  .children {
    @include apply-theme-b;
  }
}
```

### Overrides

Various components can override their theme styles via props e.g. Buttons and Tags.
